# Sicurezza

Grazie per aiutarci a mantenere sicuro questo progetto.

## Segnalazione vulnerabilità
- Non aprire issue pubbliche per vulnerabilità.
- Invia una segnalazione privata a SECURITY_CONTACT (imposta il contatto in questo file o in repo settings).
- Includi: versione/commit, passi per riprodurre, impatto, suggerimenti di fix.

## Gestione segreti
- Non committare segreti. Usa `.env` e i secret di CI.
- Rotazione: se un segreto trapela, ruotalo immediatamente e invalida i token.

## Dipendenze
- Mantieni aggiornate le dipendenze e attiva security alerts.
- Valuta l’impatto di licenze/fornitori.

## Policy di disclosure
- Confermeremo la ricezione entro 3 giorni lavorativi.
- Forniremo uno stato iniziale entro 7 giorni.
